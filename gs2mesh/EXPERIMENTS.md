## 實驗 1 — naive Poisson（高斯中心點）❌ 2026-07-17

- **方法**:取高斯中心當點雲 → open3d `estimate_normals` → Poisson depth=10。
- **平台**:Colab Pro+ / A100。
- **結果**:490k 頂點 / 964k 面 / bbox 對 / 94% 單一主殼 / winding 一致。
- **目視**:❌ 坑疤塊狀,無平牆平地。`screenshots/2026-07-17_naive-poisson_{iso,side}.png`。
- **根因**:只用中心點,**丟掉每個高斯的異向協方差(scale+rot)**——表面朝向資訊全失,
  Poisson 從無向點雲硬包等值面,必然疙瘩。
- **下一步**:實驗 2 用協方差算有向法線,直接修這個根因。

## 實驗 2 — 有向圓盤法線 Poisson（post-hoc,用協方差）⏳ 進行中

- **假設**:每個 3D 高斯的**最扁軸 = 表面法線**（2DGS 的核心洞見,但這裡 post-hoc 套在
  已訓練的 ply 上,不重訓）。給 Poisson 真實有向法線,應比實驗 1 平整得多。
- **方法**:讀 ply 的 scale_0/1/2 + rot_0/1/2/3(四元數)→ 每高斯的旋轉矩陣 →
  最小 scale 軸的方向 = 法線 → 濾透明/大高斯 → Poisson。
- **平台**:本機（124k 點,沒 segfault）。
- **法線品質**:算出的有向法線 **71% 對齊三主軸**（Z 21.5%/X 21.4%/Y 28.4%）——
  法線本身是對的,扁高斯確實貼牆/地。
- **結果**:506k 頂點 / 997k 面 / 94.4% 主殼 / bbox 8.98×3.12×13.12m。
- **目視**:❌ 仍坑疤,甚至比實驗 1 更海綿狀。`screenshots/2026-07-17_exp2-oriented_{iso,side}.png`。
- **關鍵結論(比實驗 1 更深的教訓)**:法線不是根因。真正原因是
  **3DGS 高斯中心不落在表面上**——它們散在表面周圍幾公分厚的殼裡(訓練時只被
  要求「渲染起來對」,沒被約束「位置在面上」)。中心不在面上,法線再準,等值面就是凹凸。
  扁度篩又讓覆蓋掉到 11.2%,Poisson 大幅內插 → 更海綿。
- **這排除了整個「post-hoc 用中心點」路線**（實驗 1+2 兩次證明）。正解只剩兩條:
  1. **渲染深度圖 TSDF 融合**:用已訓練 3DGS 在 726 個 COLMAP 相機位算深度圖,
     再 TSDF 融合(這才是 gs2mesh 類方法的真做法,**不必重訓**,但需 CUDA rasterizer)。
  2. **官方 2DGS 重訓**:訓練時就把高斯壓扁+拉到表面+從渲染深度取網格。最重。
- **下一步**:實驗 3 走「深度圖 TSDF」——比重訓輕,用現有 ply+COLMAP,是正解裡的省力路。
